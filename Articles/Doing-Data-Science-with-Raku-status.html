Raku Data Science & Machine Learning Ecosystem Status – November 2025
Feature Comment Pre-2021 2025 Status Essential Current Rating Key References
Data ingestion That is fundamental and all programming systems have such functionalities to various degrees. Robust JSON, CSV, CBOR files ingestion; XML and other formats can be ingested, but not in a robust manner. Various (improved) packages for JSON, CSV, markup images, PDF, etc. Umbrella ingestion function for them. ✓ ★★★ (2.5)
  • Data::Importers
  • Data::Translators
  • Omni-slurping with LLMing
  • [AA3]
Data wrangling facilitation Slicing, splitting, combining, aggregating, summarizing data can be difficult and time consuming. No serious efforts, especially, in terms of streamlining data wrangling workflows. Two major efforts for streamlining data wrangling workflows one using "pure" Raku (good for exploration) and other interfaces "outside" systems. ✓ ★★★ (3.25)
  • Data::Reshapers
  • Dan
  • Introduction to data wrangling with Raku
  • [AA3]
Statistics for data exploration This includes descriptive statistics (mean, median, 5-point summary), summarization, outlier identification, and statistical distribution functions. Various attempts, some are basic and "plain-Raku" (e.g. Stats), some connect to GSL (and do not work on macOS). Couple of major efforts exist, one is all-in-one package, the other is spread out in various packages. ✓ ★★★ (2.5)
  • Statistics
  • Statistics::Distributions
  • Statistics::OutlierIdentifiers
  • Data::Summarizers
  • Data::TypeSystem
Machine Learning algorithms
(both unsupervised and supervised)		 A few packages for doing unsupervised Machine Learning (like Text::Markov). At least one supervised ML package connecting to external systems, a set of unsupervised ML packages for clustering, association rules, fitting, tries with frequencies, and Recommendation Systems (RS). RS and tries can be used as classifiers. ✓ ★★★ (2.5)
  • Algorithm::XGBoost
  • ML::* packages
Data visualization facilitation Insightful plots over data are used in Data Science most of the time. A few small packages for plotting, at least one connecting external systems (like GnuPlot), none of them that useful for Data Science. Two "solid" packages for Data Science visualizations: JavaScript::D3 and JavaScript::Google::Charts; also Text::Plot for ASCII plots. ✓ ★★★★
  • JavaScript::D3
  • JavaScript::Google::Charts
  • Text::Plot
  • The Raku-ju hijack hack for D3.js
  • Geographics data in Raku demo
Interactive computing environment(s) Any data exploration is done in interactive manner with multiple changes of the data, and analysis or pattern finding workflows. The (basic) Raku REPL, related Emacs major-mode, and the notebook environment Jupyter::Kernel. In addition: RakuMode for Wolfram Notebooks, Jupyter::Chatbook for seamless integration with LLMs. ✓ ★★★★
  • Connecting Mathematica and Raku
  • Exploratory Data Analysis with Raku
Literate programming Literate Programming is very important for Data Science because of the needs for Reproducible Research. None, except Jupyter::Kernel, but that not useful because of the lack of good graphics. Multiple Literate Programming solutions are in place. Because of the good graphics and LLM support in Raku, LP is particularly powerful. ✓ ★★★★★ (4.5)
  • Notebook transformations
Data generation and retrieval For didactical and development purposes random data generation and retrieval of well known dataset is needed. — Dedicated packages for random data, example datasets, and geographic data. ★★★★ (3.5)
  • Data::Generators
  • Data::ExampleDatasets
  • Data::Geographics
  • Geographics data demo
External Data Science (DS) and Machine Learning (ML) orchestration Effective way to do DS and ML and easily move the developed computations to other systems. Allows reuse and having confidence that the utilized algorithms are properly implemented and fast. Various projects connecting to database systems (e.g. MySQL). Several orchestration tools available (Dan, ZMQ, WolframAlpha, H2O). ★★★ (2.5)
  • Dan
  • Proc::ZMQed
  • WWW::WolframAlpha
  • H2O::Client
Interactive interfaces to parameterized workflows (dashboards) Very useful for getting data insights by dynamically changing different statistics based on parameters. -- Early effort with Air::Examples + HTMX + Cro; Google Charts can be extended for interactive controls. ★
  • Air::Examples
  • JavaScript::Google::Charts
